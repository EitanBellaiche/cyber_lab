MITM Demo Guide

Machine IP on Wi-Fi:
10.100.102.25

Important:
- Do not use localhost for the MITM demo.
- Use the machine IP from another device on the same Wi-Fi network.
- On port 8080 use HTTP only.
- On port 8443 use HTTPS only.

1. Vulnerable server demo

Run:
sudo ./zookld --vulnerable

From the phone or another computer on the same Wi-Fi, browse to:
http://10.100.102.25:8080/zoobar/index.cgi/

In Wireshark:
- Capture on interface: wlp2s0
- Filter: tcp.port == 8080

Expected result:
- You should see HTTP traffic.
- If you log in, username and password can appear in plaintext.

Important:
- Type the URL with http:// explicitly.
- Do not open https://10.100.102.25:8080 because port 8080 is not TLS.

2. Protected server demo

Run:
sudo ./zookld

From the phone or another computer on the same Wi-Fi, browse to:
https://10.100.102.25:8443/zoobar/index.cgi/

In Wireshark:
- Capture on interface: wlp2s0
- Filter: tcp.port == 8443 || tls

Expected result:
- You should see TLS traffic, not plaintext HTTP credentials.
- The browser may show a certificate warning if the local CA is not trusted on that device.

If the browser warns:
- For demo purposes, continue to the site if your browser allows it.
- The traffic is still encrypted even if the certificate is not trusted.

3. If the phone/browser fails on the vulnerable server

If zookd-vulnerable prints:
Cannot parse HTTP request (1)

This usually means the phone tried HTTPS against the HTTP server.

Use only:
http://10.100.102.25:8080/zoobar/index.cgi/

not:
https://10.100.102.25:8080/zoobar/index.cgi/

4. Summary

Vulnerable:
http://10.100.102.25:8080/zoobar/index.cgi/

Protected:
https://10.100.102.25:8443/zoobar/index.cgi/
